Ubuntu 24.04 に rclone をインストールして CLI で pCloud を操作する

Published: 2025-06-08
  • #Ubuntu
  • #rclone
  • #pCloud

概要

環境

  • Windows 11 (クライアント)
  • Ubuntu 24.04 (サーバ/rclone インストール先)
  • rclone 1.69.3
  • pCloud

手順

事前確認

  • Ubuntu Server は CLI で運用しているため、クライアントから SSH 接続して作業する。
  • rclone に pCloud を設定するためには、ブラウザでの認証が必要となる。
  • 今回の手順では、SSH のローカルポートフォワーディングを使用して、クライアント側のブラウザで認証する。

インストール

Ubuntu に rclone をインストールする。インストールは以下を参照した。

  • Install

次のコマンドを実行し、rclone をインストールする。

sudo -v ; curl https://rclone.org/install.sh | sudo bash
rclone v1.69.3 has successfully installed.
Now run "rclone config" for setup. Check https://rclone.org/docs/ for more details.

既にインストール済みの場合は、次のコマンドを実行し、rclone を最新の安定板にアップデートする。

rclone selfupdate --stable

設定

rclone に pCloud を設定する。設定は以下を参照した。

  • pCloud

次のコマンドを実行する。

rclone config

対話式の設定プロセスが表示されるため、指示に従い設定する。

No remotes found, make a new one?
n) New remote
s) Set configuration password
q) Quit config
n/s/q> n
name> pcloud

n/s/q> に n を入力する。

name> には、リモート接続先を識別する任意の名前を指定する。今回は pcloud を入力する。

Type of storage to configure.
Choose a number from below, or type in your own value
[snip]
XX / Pcloud
   \ "pcloud"
[snip]
Storage> pcloud

Storage> に pCloud の数値 XX を入力する。(今回のインストール時は 40 だった。)

Pcloud App Client Id - leave blank normally.
client_id>
Pcloud App Client Secret - leave blank normally.
client_secret>

client_id>, client_secret> は空のまま進める。

Edit advanced config?
y) Yes
n) No (default)
y/n> n

y/n> は n を入力する。

Use web browser to automatically authenticate rclone with remote?
 * Say Y if the machine running rclone has a web browser you can use
 * Say N if running rclone on a (remote) machine without web browser access
If not sure try Y. If Y failed, try N.
y) Yes
n) No
y/n> y

y/n> に y を入力する。ブラウザが利用できない環境の場合は n を入力し、ブラウザが利用できる別の環境の rclone を使って認証する必要がある。(詳細は Remote Setup を参照する。)

If your browser doesn't open automatically go to the following link: http://127.0.0.1:53682/auth
Log in and authorize rclone for access
Waiting for code...

今回は CLI のみの Ubuntu Server に SSH で接続しているため、クライアント側のブラウザでアクセスする必要がある。

クライアント側にて、以下コマンドでローカルポートをフォワードする。(user@server は接続先のユーザとホスト名を指定する。)

ssh -L 53682:127.0.0.1:53682 user@server

クライアント側のブラウザで http://127.0.0.1:53682/auth を開き、認証する。

Got code
Configuration complete.
Options:
- type: pcloud
- eapi.pcloud.com
- token: {"access_token":"XXX","token_type":"bearer","expiry":"0001-01-01T00:00:00Z"}
Keep this "pcloud" remote?
y) Yes this is OK
e) Edit this remote
d) Delete this remote
y/e/d> y

y/e/d> に y を入力する。

Current remotes:

Name                 Type
====                 ====
pcloud               pcloud

e) Edit existing remote
n) New remote
d) Delete remote
r) Rename remote
c) Copy remote
s) Set configuration password
q) Quit config
e/n/d/r/c/s/q> q

Current remotes に pcloud が表示される。

e/n/d/r/c/s/q> に q を入力する。

確認

以下コマンドにて、pCloud のルートディレクトリのリストを表示する。(pcloud 部分は config で設定した name の値)

rclone lsd pcloud:

以下コマンドにて、pCloud のルートディレクトリのファイルを表示する。

rclone ls pcloud:

以下コマンドにて、ローカルの /home/source ディレクトリ を pCloud の backup ディレクトリにバックアップする。

rclone copy /home/source pcloud:backup
