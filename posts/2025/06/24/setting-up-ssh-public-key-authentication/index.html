SSH の公開鍵認証を設定する

概要#

  • Windows 端末から Ubuntu 端末に SSH 接続するとき、公開鍵で認証できるようにする。

環境#

  • Windows 11 (接続元)
    • OpenSSH_for_Windows_9.5p1, LibreSSL 3.8.2
  • Ubuntu 24.04 LTS (接続先)
    • OpenSSH_9.6p1 Ubuntu-3ubuntu13.12, OpenSSL 3.0.13 30 Jan 2024

手順#

以下を例に説明する。

  • 接続ユーザ名: your
  • 接続先ホスト名: example.local

鍵の作成#

Windows にて、公開鍵と秘密鍵を作成する。

ユーザプロファイル配下の .ssh フォルダに移動する。.ssh フォルダがなければ作成する。

cd $env:USERPROFILE\.ssh

次のコマンドを実行し、公開鍵と秘密鍵を作成する。

ssh-keygen -t ed25519

Generating public/private ed25519 key pair.
Enter file in which to save the key (C:\Users\your/.ssh/id_ed25519): id_ed25519_example
Enter passphrase (empty for no passphrase):
Enter same passphrase again:
Your identification has been saved in id_ed25519_example
Your public key has been saved in id_ed25519_example.pub
The key fingerprint is:
SHA256:XXX your@example.local
The key's randomart image is:
+--[ED25519 256]--+
|                 |
|                 |
|                 |
|                 |
|                 |
|                 |
|                 |
|                 |
|                 |
+----[SHA256]-----+

ここではファイル名を id_ed25519_example とし、パスフレーズは未入力としている。

.ssh 配下に以下の 2 ファイルが作成されたことを確認する。

  • 秘密鍵: id_ed25519_gamma
  • 公開鍵: id_ed25519_gamma.pub

公開鍵の登録#

作成した公開鍵を接続先ホストに登録する。

接続元ホストが Windows の場合は ssh-copy-id コマンドが使えないため、PowerShell にて以下を実行する。

$ssh_user = "your"
$ssh_host = "example.local"
$ssh_port = "22"
$identityFile = "$env:USERPROFILE\.ssh\id_ed25519_example.pub"
$pubkey = Get-Content $identityFile -Raw
ssh -p $ssh_port "$ssh_user@$ssh_host" "mkdir -p ~/.ssh && chmod 700 ~/.ssh && echo '$pubkey' >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"

接続先ホストの ~/.ssh/authorized_keys に接続元ホストの公開鍵情報が登録されていれば良いため、他の方法で接続先ホストに接続して登録しても問題ない。

接続情報の設定#

接続元ホストにて、任意のエディタで ~/.ssh/config を開く。

以下を入力し、保存する。

Host example.local
  User your
  Hostname example.local
  IdentityFile ~/.ssh/id_ed25519_example

この設定を追加することで、接続時に鍵の指定が必要なくなる。

接続確認#

接続元ホストにて、以下のコマンドを実行する。

ssh your@example.local

パスワードを聞かれることなく公開鍵認証で接続できることを確認する。
