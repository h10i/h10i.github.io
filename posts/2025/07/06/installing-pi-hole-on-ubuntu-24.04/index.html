Ubuntu 24.04 に Pi-hole をインストールする

概要#

  • Pi-hole とは、ネットワークの通信を DNS レベルでブロックできる広告ブロッカーである。

環境#

  • Ubuntu 24.04 LTS
  • Pi-hole
    • Core version is v6.1.2 (Latest: v6.1.2)
    • Web version is v6.2.1 (Latest: v6.2.1)
    • FTL version is v6.2.3 (Latest: v6.2.3)

手順#

インストール#

以下のコマンドを実行する。

curl -sSL https://install.pi-hole.net | bash

確認#

以下のコマンドを実行し、サービスが正常に稼働していることを確認する。

sudo systemctl status pihole-FTL

DNS の設定#

クライアント端末にて、優先 DNS に Pi-hole をインストールしたサーバの IP を設定する。

全ての端末に設定したい場合は、ルータの設定より、配布する優先 DNS と代替 DNS を変更する。

Web UI のポート変更#

デフォルトでは 80 ポートでサービスが提供されているが、他サービスで既に使っているため、別のポートに変更する。

任意のエディタで /etc/pihole/pihole.toml を開く。

sudo vi /etc/pihole/pihole.toml

80o の部分を 82o に変更する。

port = "82o,443os,[::]:82o,[::]:443os" ### CHANGED, default = "80o,443os,[::]:80o,[::]:443os"

pihole-FTL サービスを再起動する。

sudo systemctl restart pihole-FTL

内部では 82 ポートを使うが、外部からの利用は 80 で接続したいため、nginx でフォーワードする。

    location /admin/ {
        proxy_pass http://localhost:82/admin/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /api/ {
        proxy_pass http://localhost:82/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

nginx サービスを再起動する。

sudo nginx -t && sudo systemctl reload nginx

http://{server}/admin/ に接続し、ログインできることを確認する。

DNS レコードの設定#

Web UI より、SYSTEM > Settings > Local DNS Records を選択する。

適当なドメインと IP を追加する。

  • Domain: example.com
  • Associated IP: 192.168.1.123

クライアント端末にて、指定した IP に解決されることを確認する。

ping example.com
