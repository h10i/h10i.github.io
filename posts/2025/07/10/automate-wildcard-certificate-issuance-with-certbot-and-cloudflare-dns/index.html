Cloudflare DNS を使ってワイルドカード証明書を Certbot で自動発行する

概要#

  • HTTPS 接続するために必要な独自ドメインの証明書を Certbot で発行し、自動で更新されるようにする。

環境#

  • Ubuntu 24.04 LTS
  • certbot 4.1.1
  • nginx 1.26.2

手順#

説明のためドメインは example.com を指定しているが、実際のドメインに置き換えて実施する。

Certbot のインストール#

次のコマンドを実行し、Certbot をインストールする。

sudo snap install --classic certbot

次のコマンドを実行し、certbot を実行できるようにする。

sudo ln -s /snap/bin/certbot /usr/bin/certbot

次のコマンドを実行し、ルート権限で実行されるプラグインを通じてリソースにアクセスできるように設定する。

sudo snap set certbot trust-plugin-with-root=ok

次のコマンドを実行し、Cloudflare のプラグインをインストールする。

sudo snap install certbot-dns-cloudflare

API キーの設定#

Cloudflare の My Profile > API Tokens より、Create Token をクリックする。

  • API Tokens ｜ Cloudflare

Edit zone DNS の Use template をクリックする。

以下を設定し、Continue to summary をクリックする。({your domain} には、対象となるドメインを指定する。)

  • Permissions
    • Zone:DNS:Edit
  • Zone Resources
    • Include:Specific zone:example.com
  • Client IP Address Filtering
    • セキュリティに応じて適切に設定する。
  • TTL
    • セキュリティに応じて適切に設定する。

Edit zone DNS API token summary にて、内容を確認し、Create Token をクリックする。

次のコマンドを実行し、表示されたトークンが使用できることを確認する。

curl "https://api.cloudflare.com/client/v4/user/tokens/verify" \
     -H "Authorization: Bearer 0123456789abcdef0123456789abcdef01234567"

任意のエディタで /etc/letsencrypt/cloudflare.ini を作成し、トークンの値を保存する。

sudo mkdir -p /etc/letsencrypt
sudo vi /etc/letsencrypt/cloudflare.ini

# Cloudflare API token used by Certbot
dns_cloudflare_api_token = 0123456789abcdef0123456789abcdef01234567

次のコマンドを実行し、作成したファイルの権限を設定する。

sudo chmod 0400 /etc/letsencrypt/cloudflare.ini

証明書の発行#

次のコマンドを実行し、証明書を発行する。(--dry-run を付けると、実際に発行せず挙動を確認できる。)

sudo certbot certonly \
  --dns-cloudflare \
  --dns-cloudflare-credentials /etc/letsencrypt/cloudflare.ini \
  --dns-cloudflare-propagation-seconds 60 \
  -d example.com \
  -d *.example.com

証明書の再発行#

以下のコマンドを実行し、証明書を再発行する。(--dry-run を付けると、実際に発行せず挙動を確認できる。)

sudo certbot renew

systemd による自動再発行#

次のコマンドを実行し、タイマーの一覧を出力する。

sudo systemctl list-timers

Tue 2025-07-08 09:28:00 JST 16h - - snap.certbot.renew.timer snap.certbot.renew.service

snap.certbot.renew.timer がインストールされていることを確認する。

無効化されている場合は、次のコマンドを実行し、有効化する。

sudo systemctl enable snap.certbot.renew.timer

詳細な実行タイミングは以下を確認し、必要に応じて編集する。

sudo less /etc/systemd/system/snap.certbot.renew.timer

スケジュールを変更した場合、次のコマンドを実行して反映させる。

sudo systemctl daemon-reload

nginx の HTTPS 設定#

ssl_certificate と ssl_certificate_key に発行した証明書と鍵のパスを設定する。

server {
  listen 443 ssl;
  server_name test.example.com;
  ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
  ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

  ...
}

次のコマンドを実行し、適用する。

sudo nginx -t | sudo systemctl reload nginx

https://test.example.com に接続し、HTTPS でエラーなく接続できることを確認する。
