systemd のサービス実行結果を Discord に通知する

概要#

  • systemd のサービス実行結果を Discord に通知する。

環境#

  • Ubuntu 24.04 LTS
  • Discord

手順#

Discord の Webhook URL 取得#

Discord にて、適当なサーバとチャンネルを作成する。

チャンネルの設定より、Integrations > Webhooks を開く。

New Webhook をクリックし、新しい Webhook を作成する。

Copy Webhook URL をクリックする。

次のコマンドを実行し、通知が飛ぶことを確認する。

curl -X POST \
  -H "Content-Type: application/json" \
  -d '{"content": "Hello, Discord!"}' \
  https://discord.com/api/webhooks/{webhook.id}/{webhook.token}

通知スクリプト#

/opt/notify-discord/bin/notify-discord.sh を作成する。

#!/bin/bash

set -euo pipefail

APP_NAME="$1"
SERVICE_NAME="${APP_NAME}.service"
CONFIG_DIR="/etc/notify-discord.d"
CONFIG_FILE="$CONFIG_DIR/${APP_NAME}.conf"

# default config
NOTIFY_ON_SUCCESS=false
NOTIFY_ON_FAILURE=true
SUCCESS_INCLUDE_STATUS=true
SUCCESS_INCLUDE_TIMESTAMPS=false
SUCCESS_INCLUDE_LOGS=false
FAILURE_INCLUDE_STATUS=true
FAILURE_INCLUDE_TIMESTAMPS=true
FAILURE_INCLUDE_LOGS=true
DISCORD_WEBHOOK_URL=""
DEBUG=false

debug() {
    if [ "$DEBUG" = "true" ]; then
      echo "[DEBUG] $*" >&2
    fi
}

if [ -f "$CONFIG_FILE" ]; then
    source "$CONFIG_FILE"
else
    echo "No config for $SERVICE_NAME. Skipping notification." >&2
    exit 0
fi

debug "Loaded config for $SERVICE_NAME"
debug "Webhook: $DISCORD_WEBHOOK_URL"

STATE=$(systemctl is-failed "$SERVICE_NAME" || true)
SHOW=$(systemctl show "$SERVICE_NAME")
STATUS=$(echo "$SHOW" | grep '^ExecMainStatus=' | cut -d= -f2)
START_TS=$(echo "$SHOW" | grep '^ExecMainStartTimestamp=' | cut -d= -f2)
EXIT_TS=$(echo "$SHOW" | grep '^ExecMainExitTimestamp=' | cut -d= -f2)

debug "Service state: $STATE"
debug "Status code: $STATUS"
debug "Start: $START_TS"
debug "End: $EXIT_TS"

RAW_LOG=$(journalctl -u "$SERVICE_NAME" -n 10 --no-pager 2>/dev/null | tail -c 1700)

if [ "$STATE" = "failed" ] && [ "$NOTIFY_ON_FAILURE" != "true" ]; then
    debug "Failure notification disabled"
    exit 0
elif [ "$STATE" != "failed" ] && [ "$NOTIFY_ON_SUCCESS" != "true" ]; then
    debug "Success notification disabled"
    exit 0
fi

if [ "$STATE" = "failed" ]; then
    TITLE="❌ $SERVICE_NAME Failed"
    COLOR=15158332
    INCLUDE_STATUS=$FAILURE_INCLUDE_STATUS
    INCLUDE_TIMESTAMPS=$FAILURE_INCLUDE_TIMESTAMPS
    INCLUDE_LOGS=$FAILURE_INCLUDE_LOGS
else
    TITLE="✅ $SERVICE_NAME Succeeded"
    COLOR=3066993
    INCLUDE_STATUS=$SUCCESS_INCLUDE_STATUS
    INCLUDE_TIMESTAMPS=$SUCCESS_INCLUDE_TIMESTAMPS
    INCLUDE_LOGS=$SUCCESS_INCLUDE_LOGS
fi

FIELDS_ARRAY=()

# status
if [ "$INCLUDE_STATUS" = "true" ]; then
    FIELDS_ARRAY+=("$(jq -n --arg name "Exit Code" --arg value "$STATUS" --argjson inline true \
        '{name: $name, value: $value, inline: $inline}')")
fi

# timestamps
if [ "$INCLUDE_TIMESTAMPS" = "true" ]; then
    FIELDS_ARRAY+=("$(jq -n --arg name "Start" --arg value "$START_TS" --argjson inline false \
        '{name: $name, value: $value, inline: $inline}')")
    FIELDS_ARRAY+=("$(jq -n --arg name "End" --arg value "$EXIT_TS" --argjson inline false \
        '{name: $name, value: $value, inline: $inline}')")
fi

# logs
if [ "$INCLUDE_LOGS" = "true" ]; then
    FIELDS_ARRAY+=("$(jq -n --arg name "Logs" --arg value "\`\`\`log\n$RAW_LOG\n\`\`\`" --argjson inline false \
        '{name: $name, value: $value, inline: $inline}')")
fi

# join array into JSON
FIELDS_JSON=$(printf '%s\n' "${FIELDS_ARRAY[@]}" | jq -s '.')

# build JSON payload
JSON=$(jq -n \
    --arg title "$TITLE" \
    --argjson color "$COLOR" \
    --argjson fields "$FIELDS_JSON" \
    '{embeds: [{title: $title, color: $color, fields: $fields}]}' )

debug "Sending JSON payload:"
debug "$JSON"

# send to Discord
RESPONSE=$(curl -s -w "%{http_code}" -o /tmp/discord_notify_result \
  -X POST "$DISCORD_WEBHOOK_URL" \
  -H "Content-Type: application/json" \
  -d "$JSON")

HTTP_CODE="$RESPONSE"
BODY="$(cat /tmp/discord_notify_result)"
rm -f /tmp/discord_notify_result

debug "Discord HTTP status: $HTTP_CODE"
debug "Discord response body: $BODY"

if [[ "$HTTP_CODE" -ge 200 && "$HTTP_CODE" -lt 300 ]]; then
    debug "Notification sent successfully."
else
    echo "[ERROR] Failed to notify Discord: HTTP $HTTP_CODE" >&2
    echo "$BODY" >&2
    exit 1
fi

このスクリプトでは、systemctl show {service_name} でサービスの実行結果を取得し、通知メッセージに含めている。

設定はサービス毎に変更できるようにしているため、/etc/notify-discord.d 配下に {app_name}.conf を作成する。

今回は test-scripts@service 用の設定として、/etc/notify-discord.d/test-scripts.conf を作成する。

NOTIFY_ON_SUCCESS=true
NOTIFY_ON_FAILURE=true

SUCCESS_INCLUDE_STATUS=true
SUCCESS_INCLUDE_TIMESTAMPS=true
SUCCESS_INCLUDE_LOGS=false

FAILURE_INCLUDE_STATUS=true
FAILURE_INCLUDE_TIMESTAMPS=true
FAILURE_INCLUDE_LOGS=true

DISCORD_WEBHOOK_URL="https://discord.com/api/webhooks/000/xxx"

DEBUG=true

DISCORD_WEBHOOK_URL には、Discord のチャンネルで作成した Webhook URL をコピーし、設定する。

通知サービス#

通知スクリプトを呼び出すサービス/etc/systemd/system/notify-discord@.serviceを作成する。

[Unit]
Description=Notify Discord about result of %i
After=network-online.target

[Service]
Type=oneshot
ExecStart=/opt/notify-discord/bin/notify-discord.sh %i

サービス設定例#

テスト用サービスのスクリプト /opt/test-scripts/bin/test.sh を作成する。

#!/bin/bash
set -e

if [[ "$1" == "--fail" ]]; then
  echo "Simulating failure..." >&2
  exit 1
else
  echo "Simulating success..."
  exit 0
fi

テスト用スクリプトを呼び出すサービス /etc/systemd/system/test-scripts.service を作成する。

[Unit]
Description = Test service
OnSuccess=notify-discord@test-scripts.service
OnFailure=notify-discord@test-scripts.service

[Service]
Type=oneshot
ExecStart=/opt/test-scripts/bin/test.sh

OnSuccess と OnFailure にそれぞれ成功時と失敗時に呼び出すサービスを指定する。

OnSuccess=notify-discord@test-scripts.service
OnFailure=notify-discord@test-scripts.service

先ほど作成した通知サービスに、実行するサービス名を含めた形 notify-discord@{app_name}.serviceで指定する。

追加したサービスを反映させる。

sudo systemctl daemon-reload

サービス実行#

テスト用のサービスを実行する。

sudo systemctl start test-scripts.service

通知サービスの結果を確認する。

sudo systemctl status notify-discord@test-scripts.service

正しく実行されていれば、Discord のチャットに通知が飛ぶ。
