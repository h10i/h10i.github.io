Pi-hole のドメインブロック設定で VSCode の Remote SSH がエラーになる

2025-08-26 · 1 min · Me | Suggest Changes
Table of Contents
  • 概要
  • 環境
  • 調査
  • 対処

概要#

  • VSCode で自宅サーバに SSH 接続したとき、SSH 接続に成功してから数秒経過後にRemote Extension host terminated unexpectedly 3 times within the last 5 minutes というエラーが出る。
  • Restart Remote Extension Host をクリックしても、同じエラーが再発してループする。
  • Pi-hole で mobile.events.data.microsoft.com への接続をブロックしていたことが原因だった。

環境#

  • Windows 11 (クライアント側)
  • Ubuntu 24.04 LTS (サーバ側)
  • VSCode 1.103.1
    • Remote - SSH 0.120.0

調査#

VSCode にて、Ctrl + Shift + U で Output を開く。

Extension Host (Remote) を選択し、ログにエラーが出ていないか確認する。

2025-08-21 16:46:46.664 [error] Error: Hostname/IP does not match certificate's altnames: Host: mobile.events.data.microsoft.com. is not in the cert's altnames: DNS:*.h10i.dev, DNS:h10i.dev
    at Object.checkServerIdentity (node:tls:386:12)
    at TLSSocket.onConnectSecure (node:_tls_wrap:1689:27)
    at TLSSocket.emit (node:events:518:28)
    at TLSSocket._finishInit (node:_tls_wrap:1078:8)
    at ssl.onhandshakedone (node:_tls_wrap:864:12)

mobile.events.data.microsoft.com のホスト名と IP が異なるとのことなので、自宅サーバに SSH 接続し、以下のコマンドで調べる。

nslookup mobile.events.data.microsoft.com

Server:         100.100.100.100
Address:        100.100.100.100#53

Name:   mobile.events.data.microsoft.com
Address: 0.0.0.0
Name:   mobile.events.data.microsoft.com
Address: ::

0.0.0.0 が返っているため、Pi-hole でブロックされている可能性がある。

Pi-hole のログを確認したところ、ブロックされていた。

対処#

Pi-hole で mobile.events.data.microsoft.com を許可するよう設定した。

再度 SSH 接続し、エラーにならないことを確認した。

VSCode の Remote - SSH は mobile.events.data.microsoft.com と通信できないとダメなのかもしれない。
