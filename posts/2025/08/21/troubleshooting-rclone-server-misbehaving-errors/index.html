rclone sync で server misbehaving が発生したときの調査と対処

概要#

  • tailscale で自宅サーバの Pi-hole を DNS として登録後、自宅サーバで動かしていた Rclone を使った pCloud へのバックアップが失敗するようになった。
  • Pi-hole が最初の問い合わせ先になったことで、Rclone から生じる大量の DNS クエリが Pi-hole のレート制限に達して拒否されていた。
  • Pi-hole のレート制限の上限を増やすことで、エラーが発生しなくなった。

事象#

  • 自宅サーバでは、rclone sync で pCloud へ定期的にファイル等をバックアップしている。
  • 自宅サーバでは、Pi-hole を使った DNS を動かしている。
  • Tailscale にて、Global nameservers に自宅サーバの IP アドレスを追加し、Override DNS servers を設定した。
  • Tailscale の設定変更後、rclone sync が失敗するようになった。

環境#

  • Ubuntu 24.04 LTS
  • Pi-hole
    • Core version is v6.1.2 (Latest: v6.1.4)
    • Web version is v6.2.1 (Latest: v6.2.1)
    • FTL version is v6.2.3 (Latest: v6.2.3)
  • Tailscale 1.80.0

調査#

スクリプト内で実行していた Rclone のコマンドは以下の通りである。

rclone sync "$src" "$dst" --multi-thread-streams=1 --size-only --quiet

このままだと失敗した原因がわからないため、デバッグログを出力する設定を追加する。

rclone sync "$src" "$dst" --multi-thread-streams=1 --size-only --log-file="/tmp/rclone-test.log" --log-level DEBUG

実行後にログを確認する。

less /tmp/rclone-test.log

途中までは問題なく処理されていたが、途中から以下のエラーが出続けていた。

2025/08/21 13:06:49 ERROR : ...: error reading destination directory: couldn't list files: Get "https://eapi.pcloud.com/listfolder?folderid=...": dial tcp: lookup eapi.pcloud.com on 100.100.100.100:53: server misbehaving

server misbehaving とあるため、何らかの理由で eapi.pcloud.com の名前解決ができなくなったことが原因でエラーになっている。

100.100.100.100:53 は Tailscale の DNS サーバだが、Global nameservers に自宅サーバ IP を指定して上書きしているため、自宅サーバの Pi-hole に問い合わせているはず。

対処#

短時間で大量の DNS クエリが発生したことが原因であるため、Rclone の設定を見直して DNS クエリを減らすか、Pi-hole のレート制限の緩和が必要となる。

Rclone の設定#

rclone コマンドに次のオプションを追加することで、並列処理の数を減らすことができる。

  • --checkers=N
    • 並列でファイルの存在チェックや比較をするルーチンの数
    • デフォルトは 8
  • --transfers=N
    • 並列で実行するファイル転送の数
    • デフォルトは 4

処理自体が高速に実行される場合、DNS クエリ数はそこまで減らない可能性もある。また、並列処理の件数が減るため、rclone sync の処理時間は増える可能性がある。

Pi-hole の設定#

Web UI の SYSTEM > Settings > All settings にある DNS server settings (dns) に以下の設定がある。

dns.rateLimit.count=1000
dns.rateLimit.interval=60

この値を変更することで、レート制限を緩和することができる。

今回は、Pi-hole のレート制限数を増やしたところ、エラーが発生しなくなった。
