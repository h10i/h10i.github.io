Nextcloud で Too many connections エラーが出たときの対処

概要#

  • Nextcloud (Snap 版) の MySQL 設定を変更する。

環境#

  • Ubuntu 24.04 LTS (サーバ)
  • Nextcloud 31.0.6snap1
  • Windows 11 (クライアント)
  • Nextcloud デスクトップクライアント バージョン 3.17.1 (Windows)

事象#

Nextcloud のデスクトップクライアントでファイル同期したとき、複数のファイルで次のエラーが発生し、同期に失敗した。

It's taking too long to connect to the server. Please try again later. If you need help, contact your administrator.

Nextcloud のログを確認したところ、次のエラーが大量に出ていた。

Exception Failed to connect to the database: An exception occurred in the driver: SQLSTATE[HY000] [1040] Too many connections

調査#

DB へのコネクションが原因と思われる。

次のコマンドを実行し、Nextcloud が利用している MySQL に接続する。

sudo nextcloud.mysql-client -u root

次の SQL を実行し、現在のコネクション数を調べる。

SHOW GLOBAL STATUS LIKE 'Threads_%';

+-------------------+-------+
| Variable_name     | Value |
+-------------------+-------+
| Threads_cached    | 3     |
| Threads_connected | 137   |
| Threads_created   | 214   |
| Threads_running   | 2     |
+-------------------+-------+

現在のコネクション数である Threads_connected が 137 に対して、アクティブなコネクション数である Threads_running は 2 となっている。

次のコマンドを実行し、プロセスリストを確認する。

SHOW PROCESSLIST;

+-------+-----------------+-----------+-----------+---------+--------+------------------------+------------------+
| Id    | User            | Host      | db        | Command | Time   | State                  | Info             |
+-------+-----------------+-----------+-----------+---------+--------+------------------------+------------------+
|     5 | event_scheduler | localhost | NULL      | Daemon  | 587726 | Waiting on empty queue | NULL             |
| 40022 | nextcloud       | localhost | nextcloud | Sleep   |  14327 |                        | NULL             |
| 40097 | nextcloud       | localhost | nextcloud | Sleep   |  14169 |                        | NULL             |
...
| 55738 | nextcloud       | localhost | nextcloud | Sleep   |      0 |                        | NULL             |
+-------+-----------------+-----------+-----------+---------+--------+------------------------+------------------+

Sleep となっているコネクションが大量に表示された。

次のコマンドを実行し、コネクション数の最大値を確認する。

SHOW GLOBAL VARIABLES LIKE 'max_connections';

+-----------------+-------+
| Variable_name   | Value |
+-----------------+-------+
| max_connections | 151  |
+-----------------+-------+

次のコマンドを実行し、タイムアウト値を確認する。

SHOW GLOBAL VARIABLES LIKE '%timeout%';

+-----------------------------------+----------+
| Variable_name                     | Value    |
+-----------------------------------+----------+
| connect_timeout                   | 10       |
| delayed_insert_timeout            | 300      |
| have_statement_timeout            | YES      |
| innodb_flush_log_at_timeout       | 1        |
| innodb_lock_wait_timeout          | 50       |
| innodb_rollback_on_timeout        | OFF      |
| interactive_timeout               | 28800    |
| lock_wait_timeout                 | 31536000 |
| mysqlx_connect_timeout            | 30       |
| mysqlx_idle_worker_thread_timeout | 60       |
| mysqlx_interactive_timeout        | 28800    |
| mysqlx_port_open_timeout          | 0        |
| mysqlx_read_timeout               | 30       |
| mysqlx_wait_timeout               | 28800    |
| mysqlx_write_timeout              | 60       |
| net_read_timeout                  | 30       |
| net_write_timeout                 | 60       |
| replica_net_timeout               | 60       |
| rpl_stop_replica_timeout          | 31536000 |
| rpl_stop_slave_timeout            | 31536000 |
| slave_net_timeout                 | 60       |
| ssl_session_cache_timeout         | 300      |
| wait_timeout                      | 28800    |
+-----------------------------------+----------+

コネクションの最大値が 151 であり、wait_timeout が 28800 となっている。28800 秒は 8 時間なので、Sleep となったコネクションは 8 時間経たないと切断されない。

コネクションが最大値に達すると、次のコネクションが確立できずにエラーとなっているのかもしれない。

対処#

サービスの再起動#

以下コマンドを実行し、Nextcloud を再起動する。

sudo snap restart nextcloud

コネクションがリセットされるため、同期が再開する。

Sleep のコネクションが大量に残ってしまった原因はわからないため、必要であれば次のコネクション最大値やタイムアウト値の設定を変更する。

コネクション最大値とタイムアウト値の変更#

再起動でも解消せず再発する場合は、コネクションの最大値を増やし、タイムアウト値を減らす。

次のコマンドを実行し、コネクションの最大値を 300 にする。

SET GLOBAL max_connections = 300;
SHOW GLOBAL VARIABLES LIKE 'max_connections';

+-----------------+-------+
| Variable_name   | Value |
+-----------------+-------+
| max_connections | 300   |
+-----------------+-------+

次のコマンドを実行し、コネクションのタイムアウト値を 600 (10 分) にする。

SET GLOBAL wait_timeout = 600;
SHOW GLOBAL VARIABLES LIKE 'wait_timeout';

+---------------+-------+
| Variable_name | Value |
+---------------+-------+
| wait_timeout  | 600   |
+---------------+-------+

次のコマンドを実行し、MySQL のシェルから抜ける。

EXIT;

Nextcloud を再起動する。

sudo snap restart nextcloud
