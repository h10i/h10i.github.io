JUnit 5 のテストコードを分類して Gradle で実行する

概要#

  • Gradle から JUnit のテストを実行するとき、特定のグループに含まれるテストのみ実行できるようにする。
  • グループは、独自アノテーションを定義し、テストクラスにアノテーションを付けることで分類する。
  • アノテーションを付与し忘れているテストは失敗させ、いずれかのアノテーションを付与する必要がある旨をエラーログに出す。

環境#

  • Spring Boot 3.4.5
  • Kotlin 2.0.21
  • JUnit 5

手順#

分類用アノテーションの作成#

Unit Test、Integration Test、E2E Test の 3 種類に対応するアノテーションを作成する。

object TestTags {
    const val UNIT = "unit"
    const val INTEGRATION = "integration"
    const val E2E = "e2e"
}

@Target(AnnotationTarget.CLASS)
@Retention(AnnotationRetention.RUNTIME)
@Tag(TestTags.UNIT)
annotation class UnitTest

@Target(AnnotationTarget.CLASS)
@Retention(AnnotationRetention.RUNTIME)
@Tag(TestTags.INTEGRATION)
annotation class IntegrationTest

@Target(AnnotationTarget.CLASS)
@Retention(AnnotationRetention.RUNTIME)
@Tag(TestTags.E2E)
annotation class E2ETest

JUnit 5 の Extension の設定#

アノテーションをつけ忘れた場合、テストを失敗させ、いずれかのアノテーションを付与するようログに表示させたい。そのための拡張機能を作成する。

テストクラスは以下のいずれかのアノテーションを付与しなければエラーになるようにする。

  • @UnitTest
  • @IntegrationTest
  • @E2ETest

src/test/kotlin/com/example/app/support/extension/RequiredTestTypeAnnotationCondition.kt を作成する。

package com.example.app.support.extension

import ...

class RequiredTestTypeAnnotationCondition : ExecutionCondition {
    private val allowedAnnotations: Set<KClass<out Annotation>> = setOf(
        UnitTest::class,
        IntegrationTest::class,
        E2ETest::class,
    )

    override fun evaluateExecutionCondition(context: ExtensionContext): ConditionEvaluationResult {
        val clazz = context.testClass.orElse(null)
            ?: return ConditionEvaluationResult.enabled("Not a test class")

        if (hasAllowedAnnotationRecursively(clazz)) {
            return ConditionEvaluationResult.enabled("Valid test type annotation present.")
        }

        throw IllegalStateException(
            """

            ❌ Test class '${clazz.name}' is missing a required test type annotation.

              ➤ One of the following annotations must be present on the class:
                 ${allowedAnnotations.joinToString("\n                 ") { "@${it.simpleName}" }}

              ➤ Example usage:
                 @UnitTest
                 class ${clazz.simpleName} { ... }

            """.trimIndent()
        )
    }

    private fun hasAllowedAnnotationRecursively(start: Class<*>): Boolean {
        var current: Class<*>? = start
        while (current != null) {
            if (allowedAnnotations.any { current.isAnnotationPresent(it.java) }) return true
            current = current.enclosingClass
        }
        return false
    }
}

src/test/resources/META-INF/services/org.junit.jupiter.api.extension.Extension を作成する。

com.example.app.support.extension.RequiredTestTypeAnnotationCondition

src/test/resources/junit-platform.properties を作成し、拡張機能の自動検出を有効化する。

junit.jupiter.extensions.autodetection.enabled=true

Gradle の設定#

テスト実行時に、includeTags プロパティでテスト対象のタグ名をカンマ区切りで指定できるようにする。

tasks.withType<Test> {
    useJUnitPlatform {
        if (project.hasProperty("includeTags")) {
            @Suppress("SpreadOperator")
            includeTags(*project.property("includeTags").toString().split(",").toTypedArray())
        }
    }
}

テストの実行#

例えば、Unit Test と Integration Test のみを実行したい場合は、includingTags に unit と integration を , 区切りで指定する。

gradlew test -PincludeTags="unit,integration"

特に指定しない場合、全てのテストを実行する。

gradlew test

ExampleTest クラスにて、アノテーションをつけ忘れた状態でテストを実行した場合、次のエラーが出て失敗する。

❌ Test class 'com.example.app.ExampleTest' is missing a required test type annotation.

  ➤ One of the following annotations must be present on the class:
     @UnitTest
     @IntegrationTest
     @E2ETest

  ➤ Example usage:
     @UnitTest
     class ExampleTest { ... }
