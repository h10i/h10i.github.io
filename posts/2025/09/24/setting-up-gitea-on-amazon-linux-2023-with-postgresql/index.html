Amazon Linux 2023 上に PostgreSQL を使用した Gitea を構築する

概要#

  • Amazon Linux 2023 上に Gitea を構築する。
  • DB は PostgreSQL を使用する。

環境#

  • Amazon Linux 2023
  • PostgreSQL 17
  • Gitea 1.24.6

手順#

EC2#

  1. Amazon Linux 2023 の EC2 を起動する。

  2. Amazon Linux 2023 に SSH 接続する。(amazon-linux-gitea.pem は作成したキーファイル)

    ssh -i "amazon-linux-gitea.pem" ec2-user@x.x.x.x

  3. パッケージをアップデートする。

    sudo dnf -y update

Git#

  1. Amazon Linux 2023 に SSH 接続する。

  2. Git をインストールする。

    sudo dnf -y install git

PostgreSQL#

  1. PostgreSQL をインストールする。

    sudo dnf install -y postgresql17-server postgresql17-contrib

  2. PostgreSQL を初期化する。

    sudo postgresql-setup --initdb

  3. pg_hba.conf を変更する。

    sudo cp /var/lib/pgsql/data/postgresql.conf{,.bk}
sudo vi /var/lib/pgsql/data/postgresql.conf
sudo diff -u --color /var/lib/pgsql/data/postgresql.conf{.bk,}

     --- /var/lib/pgsql/data/postgresql.conf.bk      2025-09-17 02:04:54.749511799 +0000
 +++ /var/lib/pgsql/data/postgresql.conf 2025-09-17 02:05:32.219868946 +0000
 @@ -94,7 +94,7 @@
  # - Authentication -

  #authentication_timeout = 1min         # 1s-600s
 -#password_encryption = scram-sha-256   # scram-sha-256 or md5
 +password_encryption = scram-sha-256    # scram-sha-256 or md5
  #scram_iterations = 4096

  # GSSAPI using Kerberos

  4. PostgreSQL サービスを有効化し、開始する。

    sudo systemctl enable postgresql
sudo systemctl start postgresql

  5. PostgreSQL にログインする。

    sudo su -c "psql" - postgres

  6. gitea ロールを作成する。

    CREATE ROLE gitea WITH LOGIN PASSWORD 'gitea';

  7. giteadb データベースを作成する。

    CREATE DATABASE giteadb WITH OWNER gitea TEMPLATE template0 ENCODING UTF8 LC_COLLATE 'en_US.UTF-8' LC_CTYPE 'en_US.UTF-8';

  8. pg_hba.conf を変更する。

    sudo cp /var/lib/pgsql/data/pg_hba.conf{,.bk}
sudo vi /var/lib/pgsql/data/pg_hba.conf
sudo diff -u --color /var/lib/pgsql/data/pg_hba.conf{.bk,}

     --- /var/lib/pgsql/data/pg_hba.conf.bk  2025-09-17 01:13:03.749791913 +0000
 +++ /var/lib/pgsql/data/pg_hba.conf     2025-09-17 02:23:00.209880393 +0000
 @@ -110,11 +110,12 @@
  # TYPE  DATABASE        USER            ADDRESS                 METHOD

  # "local" is for Unix domain socket connections only
 +local   giteadb         gitea                                   scram-sha-256
  local   all             all                                     peer
  # IPv4 local connections:
 -host    all             all             127.0.0.1/32            ident
 +host    all             all             127.0.0.1/32            scram-sha-256
  # IPv6 local connections:
 -host    all             all             ::1/128                 ident
 +host    all             all             ::1/128                 scram-sha-256
  # Allow replication connections from localhost, by a user with the
  # replication privilege.
  local   replication     all                                     peer

  9. PostgreSQL を再起動する。

    sudo systemctl restart postgresql

  10. 接続テストする。

    psql -U gitea -d giteadb

Gitea#

  1. 以下をダウンロードする。(Amazon Linux 2023 から直接ダウンロードしても可)

    • https://dl.gitea.com/gitea/1.24.6/gitea-1.24.6-linux-amd64

  2. ダウンロードしたバイナリを SCP で転送する。

    scp -i "amazon-linux-gitea.pem" "gitea-1.24.6-linux-amd64" ec2-user@x.x.x.x:

  3. Amazon Linux 2023 に SSH 接続する。

  4. Gitea のバイナリの名称を変更する。

    mv gitea-1.24.6-linux-amd64 gitea

  5. Gitea のバイナリに実行権限を付与する。

    chmod +x gitea

  6. git ユーザを作成する。

    sudo groupadd --system git
sudo adduser \
   --system \
   --shell /bin/bash \
   --comment 'Git Version Control' \
   --gid git \
   --home-dir /home/git \
   --create-home \
   git

  7. 必要なディレクトリを作成する。

    sudo mkdir -p /var/lib/gitea/{custom,data,log}
sudo chown -R git:git /var/lib/gitea/
sudo chmod -R 750 /var/lib/gitea/
sudo mkdir /etc/gitea
sudo chown root:git /etc/gitea
sudo chmod 770 /etc/gitea

  8. バイナリをコピーする。

    sudo cp gitea /usr/local/bin/gitea

  9. サービスを作成する。

    sudo tee /etc/systemd/system/gitea.service<<EOF
[Unit]
Description=Gitea (Git with a cup of tea)
After=network.target
Wants=postgresql.service
After=postgresql.service

[Service]
# Uncomment the next line if you have repos with lots of files and get a HTTP 500 error because of that
# LimitNOFILE=524288:524288
RestartSec=2s
Type=simple
User=git
Group=git
WorkingDirectory=/var/lib/gitea/
ExecStart=/usr/local/bin/gitea web --config /etc/gitea/app.ini
Restart=always
Environment=USER=git HOME=/home/git GITEA_WORK_DIR=/var/lib/gitea

[Install]
WantedBy=multi-user.target
EOF

  10. サービスを開始する。

    sudo systemctl daemon-reload
sudo systemctl enable gitea
sudo systemctl start gitea

Nginx#

  1. Nginx をインストールする。

    sudo dnf install -y nginx

  2. Gitea 用の設定を追加する。(ドメインは Route 53 にて、gitea.example.internal に x.x.x.x を設定した A レコードを作成しておく。)

    sudo tee /etc/nginx/conf.d/gitea.conf > /dev/null <<EOF
server {
    listen 80;
    server_name gitea.example.internal;

    location / {
        proxy_pass http://localhost:3000;
    }
}
EOF

  3. Nginx の自動起動を設定する。

    sudo systemctl enable nginx

  4. Nginx を再起動する。

    sudo systemctl enable nginx
