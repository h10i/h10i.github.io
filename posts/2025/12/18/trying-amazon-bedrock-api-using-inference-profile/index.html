Amazon Bedrock の API で推論プロファイルを利用する

概要#

  • Amazon Bedrock の API を生成し、API 経由で LLM を使う。
  • 推論プロファイルを作成し、API 利用料をタグで管理する。

前提#

  • AWS CLI: 2.32.19
  • Ubuntu 24.04 LTS (AWS CLI が使えれば OS は問わない。)

手順#

AWS CLI の設定#

以下より、最新の AWS CLI をインストールする。

  • Installing or updating to the latest version of the AWS CLI - AWS Command Line Interface
snap install aws-cli --classic

aws --version

aws-cli/2.32.19 Python/3.13.11 Linux/6.14.0-37-generic exe/x86_64.ubuntu.24

リージョンを設定する。

aws configure set region ap-northeast-1

ログインする。(ブラウザが開くので、IAM ユーザで認証する。)

aws login

Attempting to open your default browser.
If the browser does not open, open the following URL:

https://ap-northeast-1.signin.aws.amazon.com/v1/authorize?...

Updated profile default to use arn:aws:iam::012345678901:user/example credentials.

認証されていることを確認する。

aws sts get-caller-identity

自分のアカウント情報が出力されることを確認する。

Bedrock の API キー生成#

IAM ユーザで AWS にサインインする。

以下の長期 API キーの生成ページを開く。

  • Amazon Bedrock ｜ ap-northeast-1

長期 API キーを生成 をクリックする。

有効期限を設定する。

生成 をクリックする。

表示された API キーをダウンロードする。

API キーの設定#

生成した API キーを環境変数に設定する。

export AWS_BEARER_TOKEN_BEDROCK={API_KEY}

推論プロファイルの作成#

自分のアカウント ID を確認する。

aws sts get-caller-identity

Account にある数字をメモする。

モデルリストを調べる。

aws bedrock list-inference-profiles \
  --type-equals SYSTEM_DEFINED \
  --region ap-northeast-1 \
  --no-cli-pager

使いたいモデルの \inferenceProfileArn` の値をメモする。

メモした情報をもとに、アプリケーション推論プロファイルを作成する。

プロファイル名は正規表現 ([0-9a-zA-Z][ _-]?)+` を満たす形式で任意に設定する。

aws bedrock create-inference-profile \
  --inference-profile-name {PROFILE_NAME} \
  --model-source copyFrom={INFERENCE_PROFILE_ARN} \
  --tags key={KEY},value={VALUE} \
  --no-cli-pager

新しい推論プロファイルが作成される。

{
    "inferenceProfileArn": "arn:aws:bedrock:ap-northeast-1:012345678901:application-inference-profile/xxxxxxxxxxxx",
    "status": "ACTIVE"
}

作成したプロファイルを削除する場合は、以下を実行する。

aws bedrock delete-inference-profile \
  --inference-profile-identifier {INFERENCE_PROFILE_ARN}

推論プロファイルの利用#

Chat / Text playground#

ブラウザで以下を開く。

  • チャット/テキストのプレイグラウンド ｜ Amazon Bedrock ｜ ap-northeast-1

モデルを選択 をクリックする。

推論プロファイル作成時に指定したモデルを選択する。(1. カテゴリ と 2. モデル を選択すると、 3. 推論 に登録した推論プロファイル名が出る。)

適用 をクリックする。

適当にチャットし、回答が表示されることを確認する。

API での利用#

AWS CLI#

以下を実行する。

aws bedrock-runtime converse \
  --model-id arn:aws:bedrock:ap-northeast-1:012345678901:application-inference-profile/xxxxxxxxxxxx \
  --messages '[{"role":"user","content":[{"text":"hello"}]}]' \
  --inference-config '{}' \
  --additional-model-request-fields '{}' \
  --region ap-northeast-1

hello に対するレスポンスが返ってくる。

{
    "output": {
        "message": {
            "role": "assistant",
            "content": [
                {
                    "text": "Hello! How can I help you today?"
                }
            ]
        }
    },
    "stopReason": "end_turn",
    "usage": {
        "inputTokens": 8,
        "outputTokens": 12,
        "totalTokens": 20,
        "cacheReadInputTokens": 0,
        "cacheWriteInputTokens": 0
    },
    "metrics": {
        "latencyMs": 1834
    }
}
curl#

以下を実行する。

curl -X POST "https://bedrock-runtime.ap-northeast-1.amazonaws.com/model/arn%3Aaws%3Abedrock%3Aap-northeast-1%3A012345678901%3Aapplication-inference-profile%2Fxxxxxxxxxxxx/converse" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $AWS_BEARER_TOKEN_BEDROCK" \
  -d '{
    "messages": [
        {
            "role": "user",
            "content": [{"text": "Hello"}]
        }
    ]
  }'

hello に対するレスポンスが返ってくる。

{
  "metrics": {
    "latencyMs": 2524
  },
  "output": {
    "message": {
      "content": [
        {
          "text": "Hello! How can I help you today?"
        }
      ],
      "role": "assistant"
    }
  },
  "stopReason": "end_turn",
  "usage": {
    "cacheReadInputTokenCount": 0,
    "cacheReadInputTokens": 0,
    "cacheWriteInputTokenCount": 0,
    "cacheWriteInputTokens": 0,
    "inputTokens": 8,
    "outputTokens": 12,
    "serverToolUsage": {},
    "totalTokens": 20
  }
}
