AtCoder の C++ 23 (gcc 12.2) ビルド環境を Docker で構築する

Published: 2024-11-10
  • #C++
  • #Podman

概要

  • AtCoder に提出する C++ 23 のソースコードをローカルで実行できる環境を Docker で構築する。

前提

  • ルール - C++入門 AtCoder Programming Guide for beginners (APG4b) の C++ 23 (gcc 12.2) にあるコンパイル・インタプリタの実行環境を構築する。

    g++-12 -std=gnu++2b -O2 -DONLINE_JUDGE -DATCODER \
-Wall -Wextra \
-mtune=native -march=native \
-fconstexpr-depth=2147483647 -fconstexpr-loop-limit=2147483647 -fconstexpr-ops-limit=2147483647 \
-I/opt/ac-library -I/opt/boost/gcc/include -L/opt/boost/gcc/lib \
-o a.out Main.cpp \
-lgmpxx -lgmp \
-I/usr/include/eigen3

環境

  • Windows 11
  • Podman version 5.2.5
  • Visual Studio Code 1.95.2
    • Dev Containers (ms-vscode-remote.remote-containers) v0.388.0
    • C/C++ Extension Pack (ms-vscode.cpptools-extension-pack) v1.3.0

調査

ルール - C++入門 AtCoder Programming Guide for beginners (APG4b) の C++ 23 (gcc 12.2) に記載されているコンパイル・インタプリタの内容より、必要なライブラリを調べる。

  • -lgmpxx -lgmp オプション
    • libgmp-dev
  • -I/opt/boost/gcc/include -L/opt/boost/gcc/lib オプション
    • libboost-system-dev
    • libboost-filesystem-dev
    • libboost-iostreams-dev
  • -I/usr/include/eigen3 オプション
    • libeigen3-dev

手順

ディレクトリ構造

プロジェクト全体のディレクトリ構造は以下とする。

workspace/
├─── .devcontainer/
│   ├── devcontainer.json
│   └── Dockerfile
│
├─── .vscode/
│   ├── c_cpp_properties.json
│   ├── launch.json
│   └── tasks.json
│
│─── ac-library/
│
└─── src/
   └── abc/
      └── abc001/
         ├── abc001_a.cpp
         ├── abc001_b.cpp
         ├── abc001_c.cpp
         └── abc001_d.cpp

これらは全てローカルに作成し、コンテナ内にマウントすることで使用する。

ac-library は、以下のコマンドで追加するか、直接取得したファイルを解凍して配置しておく。

git submodule add https://github.com/atcoder/ac-library.git ac-library

問題を解いたときのソースコードは、src 配下に作成する。この部分はのディレクトリ構造は、任意で問題ない。

コンテナ管理の設定

VSCode の Docker Path を podman に変更する。

"dev.containers.dockerPath": "podman"

Dev Containers の設定

workspace/.devcontainer/devcontainer.json を作成する。

// For format details, see https://aka.ms/devcontainer.json. For config options, see the
// README at: https://github.com/devcontainers/templates/tree/main/src/cpp
{
  "name": "C++ 23 (gcc 12.2)",
  "build": {
    "dockerfile": "Dockerfile"
  },
  "workspaceFolder": "/workspace",
  "workspaceMount": "source=${localWorkspaceFolder},target=/workspace,type=bind,consistency=cached",
  "remoteUser": "root"
}

Local にあるファイルをコンテナ内の /workspace にマウントするよう設定している。

Dockerfile の設定

workspace/.devcontainer/Dockerfile を作成する。

FROM ubuntu:24.04

RUN apt-get update \
  && apt-get install -y \
    git \
    gcc-12 \
    g++-12 \
    gdb \
    libgmp-dev \
    libboost-system-dev \
    libboost-filesystem-dev \
    libboost-iostreams-dev \
    libeigen3-dev \
  && rm -rf /var/lib/apt/lists/*

RUN update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-12 100 \
  && update-alternatives --install /usr/bin/g++ g++ /usr/bin/g++-12 100

WORKDIR /workspace

ENTRYPOINT ["/bin/bash"]

必要なライブラリのインストールと、gcc, g++ のバージョンを指定している。

コンテナの実行

VSCode にて workspace を開き、Dev Containers: Reopen in Container を実行する。

作成した Dockerfile でコンテナが作成され、コンテナ内の /workspace を開いた状態で VSCode が開く。

C/C++ 拡張機能の設定

C/C++ 拡張機能の設定 (workspace/.vscode/c_cpp_properties.json) を作成する。

{
  "configurations": [
    {
      "name": "Linux",
      "includePath": [
        "${workspaceFolder}/ac-library/",
        "${workspaceFolder}/**"
      ],
      "defines": [
        "_DEBUG",
        "UNICODE",
        "_UNICODE"
      ],
      "compilerPath": "/usr/bin/g++-12",
      "cppStandard": "c++23",
      "intelliSenseMode": "linux-gcc-x64"
    }
  ],
  "version": 4
}

ビルドタスクの作成

VSCode のビルドタスク (workspace/.vscode/tasks.json) を作成する。

{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "Build C++ File",
      "type": "shell",
      "command": "g++-12",
      "args": [
        "-std=gnu++2b",
        "-g",
        "-O0",
        "-DONLINE_JUDGE",
        "-DATCODER",
        "-Wall",
        "-Wextra",
        "-mtune=native",
        "-march=native",
        "-fconstexpr-depth=2147483647",
        "-fconstexpr-loop-limit=2147483647",
        "-fconstexpr-ops-limit=2147483647",
        "-I/workspace/ac-library",
        "-I/opt/boost/gcc/include",
        "-L/opt/boost/gcc/lib",
        "-o",
        "a.out",
        "'${file}'",
        "-lgmpxx",
        "-lgmp",
        "-I/usr/include/eigen3"
      ],
      "group": {
        "kind": "build",
        "isDefault": true
      },
      "problemMatcher": [],
      "detail": "Generated task to build C++ file"
    }
  ]
}

デバッグするため、-g オプションを追加し、 -O2 を -O0 に変更している。

また、現在開いているファイルをコンパイルするために、固定のファイル名 Main.cpp ではなく、"'${file}'" を指定している。

デバッグ設定

VSCode のデバッグ設定 (workspace/.vscode/launch.json) を作成する。

{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug C++ program",
      "type": "cppdbg",
      "request": "launch",
      "program": "${workspaceFolder}/a.out",
      "args": [
        "<",
        "input"
      ],
      "stopAtEntry": false,
      "cwd": "${workspaceFolder}",
      "environment": [],
      "externalConsole": false,
      "MIMode": "gdb",
      "setupCommands": [
        {
          "description": "Enable pretty-printing for gdb",
          "text": "-enable-pretty-printing",
          "ignoreFailures": true
        }
      ],
      "miDebuggerPath": "/usr/bin/gdb",
    }
  ]
}

確認

ビルドと実行

workspace 配下の任意の場所に test.cpp を作成する。

#include <bits/stdc++.h>
using namespace std;

int main() {
  string s;
  cin >> s;
  printf("Hello, %s!\n", s.c_str());
  return 0;
}

Ctrl + Shift + B (Tasks: Run Build Task) でビルドする。

Executing task in folder workspace: g++-12 -std=gnu++2b -g -O0 -DONLINE_JUDGE -DATCODER -Wall -Wextra -mtune=native -march=native -fconstexpr-depth=2147483647 -fconstexpr-loop-limit=2147483647 -fconstexpr-ops-limit=2147483647 -I/workspace/ac-library -I/opt/boost/gcc/include -L/opt/boost/gcc/lib -o a.out '/workspace/test.cpp' -lgmpxx -lgmp -I/usr/include/eigen3

Terminal will be reused by tasks, press any key to close it.

特にエラーがなければ、workspace/a.out が生成される。

テスト用の入力ファイル workspace/input を作成する。

John

以下のコマンドを実行する。

./a.out < input

以下が表示されれば特に問題なく実行できている。

Hello, John!

デバッグ実行

デバッグ実行したいファイルをビルドする。

ソースコードの printf の行の左側をクリックし、ブレークポイントを入れる。

左のメニューからデバッグのアイコンを選択し、Debug C++ program を実行する。

ブレークポイントで処理が止まり、変数 s の内容が表示されていれば特に問題なくデバッグ実行できている。
