Fire HD 10 (7th generation) に LineageOS 16.0 (Android 9) をインストールする

Published: 2025-05-26
  • #LineageOS
  • #Android

概要

  • 2017 年に発売した Fire HD 10 (7th generation) は Fire OS 5.7.1.0 が最新版となり、Android 5.1 がベースとなっている。
  • LineageOS 16.0 をインストールすることで、 Android 9 に対応することができる。

環境

  • Ubuntu 24.04 (作業用)
  • Fire HD 10 (Fire OS 5.7.1.0)
  • LineageOS 16.0 (Android 9)

手順

[UNLOCK][ROOT][TWRP][UNBRICK] Fire HD 10 2017 (suez) の手順を参考に進める。

事前確認

Fire HD 10 にて、次の点を確認する。

  • 必要なデータを端末外にバックアップしていること。
  • バージョンが Fire OS 5.7.1.0 であること。

ダウンロード

Ubuntu 端末にて、以下をダウンロードし、任意のパス (本手順では ~/tmp/firehd10) に配置する。。

  • LineageOS 16.0
    • lineage-16.0-20220705-UNOFFICIAL-suez.zip
  • BiTGApps arm64 9.0.0 v3.8
    • BiTGApps-arm64-9.0.0-v3.8-ROAR[1].zip
  • Dirty COW: CVE-2016-5195 Exploit
    • 20165195.zip
  • SuperSU 18+
    • SuperSU_18+.zip
  • SuperSU 2.82 SR5 APK
    • eu.chainfire.supersu_2.82-SR5-282_minAPI9(nodpi)_apkmirror.com.apk

資材の準備

  1. Ubuntu 端末にて、ダウンロードしたファイルを解凍する。(~/tmp/firehd10/exploit 配下に必要なファイルを配置する。)

    cd ~/tmp/firehd10
unzip amonet-suez-v1.1.2.zip
unzip 20165195.zip -d exploit
unzip SuperSU_18+.zip -d exploit
rm exploit/Superuser.apk
cp eu.chainfire.supersu_2.82-SR5-282_minAPI9\(nodpi\)_apkmirror.com.apk exploit/Superuser.apk

  2. Ubuntu 端末にて、以下のコマンドを実行する。

    sudo apt update
sudo add-apt-repository universe
sudo apt install python3 python3-serial adb fastboot dos2unix android-file-transfer gmtp go-mtpfs mtp-tools
mkdir ~/.android
echo 0x1949 >> ~/.android/adb_usb.ini
sudo vi /etc/udev/rules.d/51-android.rules

  3. vi (任意のエディタ) で開いた 51-android.rules に以下を入力し、上書き保存する。

    # adb protocol for blaze / ottter (Amazon Kindle Fire)
SUBSYSTEM=="usb", ATTR{idVendor}=="1949", ATTRS{idProduct}=="0006", MODE="0660", OWNER="root", GROUP="androiddev", SYMLINK+="android%n"

  4. 次のコマンドを実行する。

    sudo addgroup --system androiddev
sudo adduser ubuntu androiddev # ubuntu は作業ユーザ名
adb kill-server
adb start-server

USB デバッグの有効化

Fire HD 10 にて、以下の手順を実施する。

  1. Settings > Device にある Device Options を開く。

  2. Serial Number を 5 回タップし、開発者モードを有効化する。

  3. Settings > Device にある Device Options > Developer options を開く。

  4. Debugging にある Enable ADB を有効化する。

  5. Settings > Personal > Security & Privacy を開く。

  6. Advanced にある Apps from Unknown Sources を有効化する。

  7. Ubuntu 端末と Fire HD 10 を USB ケーブルで接続する。

  8. Ubuntu 端末にて bash を開き、次のコマンドを実行する。

    adb devices

  9. Fire HD 10 にて、Allow USB debugging? と表示されるため、Always allow from this computer にチェックを入れ、Allow を選択する。

root 化

  1. Ubuntu 端末にて、次のコマンドを実行する。

    cd ~/tmp/firehd10/exploit
adb shell mkdir /sdcard/exploit
adb push * /sdcard/exploit

adb shell
cp /sdcard/exploit/* /data/local/tmp
cd /data/local/tmp
chmod 755 *
./Matrix /data/local/tmp 2

  2. 処理が終わるまでしばらく待機し、bash に次の表示が出ることを確認する。

    <WSRoot><Exploit>0</Exploit></WSRoot>
<WSRoot><Done>0</Done></WSRoot>

  3. 次のコマンドを実行し、ルート化できていることを確認する。(シェルのユーザが root に変わっていれば OK)

    su

  4. Fire HD 10 にて、Docs アプリを開く。

  5. LOCAL STORAGE にある exploit/Superuser.apk を選択し、インストールする。

  6. SuperSU を開く。

  7. The SU binary needs to be updated. Continue ? と聞かれるため、CONTINUE を選択する。

  8. ... How would you like to install the SU binary ? と聞かれるため、NORMAL を選択する。

  9. Installation success ! と表示されたら、REBOOT を選択する。

  10. 再起動したら、SuperSU を開く。

  11. SETTINGS > Access にある Default access を選択し、Grant を設定する。

  12. Ubuntu 端末にて、次のコマンドを実行する。

    adb shell
su
cd /data/data-lib
rm -r com.wondershare.DashRoot
rm -r wondershare

TWRP のインストール

  1. Ubuntu 端末にて bash を開き、次のコマンドを実行する。

    cd ~/tmp/firehd10/amonet
sudo ./step-1.sh

  2. 以下が表示されたら YES を入力する。

    Brick preloader to continue via bootrom-exploit? (Type "YES" to continue)

  3. Ubuntu 端末と Fire HD 10 の USB ケーブルを外す。

  4. Ubuntu 端末にて、以下のコマンドを実行する。

    sudo systemctl stop ModemManager
sudo systemctl disable ModemManager
cd ~/tmp/firehd10/amonet
sudo ./bootrom-step-minimal.sh

  5. Waiting for bootrom と表示されたら、Ubuntu 端末と Fire HD 10 を USB ケーブルで接続する。

  6. Fire HD 10 の電源ボタンを 20 秒以上押す。

  7. 以下が表示されたら、Enter を押し、しばらく待機する。

    * * * Remove the short and press Enter * * *

  8. 以下が表示されたら、Enter を押し、処理が完了するまで待機する。

    If this is correct (i.e. you used "brick" option in step 1) press enter, otherwise terminate with Ctrl+C

  9. Reboot to unlocked fastboot と表示されたら処理が完了しており、Fire HD 10 が再起動する。

  10. 再起動した Fire HD 10 の左下に => HACKED FASTBOOT mode の記載があることを確認する。

  11. Ubuntu 端末にて、以下のコマンドを実行する。

    cd ~/tmp/firehd10/amonet
sudo ./fastboot-step.sh

  12. 以下が表示されたら、Enter を押し、処理が完了するまで待機する。

    Your device will be reset to factory defaults...
Press Enter to Continue...

  13. fastboot: error: Command failed と表示されるが、Fire HD 10 が再起動し、TWRP が起動していれば問題ない。

LineageOS のインストール

Ubuntu 端末と Fire HD 10 を USB で接続し、Fire HD 10 が Recovery Mode で起動した状態で以下の手順を実施する。

  1. Fire HD 10 にて、Wipe > Format Data を選択する。

  2. yes と入力し、データをフォーマットする。(完了後に Reboot System を選択しないよう注意する。)

  3. Back でメニューまで戻る。

  4. Advanced > ADB Sideload を選択する。

  5. Swipe to Start Sideload をスワイプする。

  6. Ubuntu 端末にて、以下コマンドを実行し、対象デバイスに sideload と表示されていることを確認する。

    adb devices

7.Ubuntu 端末にて、以下コマンドを実行する。

cd ~/tmp/firehd10/
adb sideload lineage-16.0-20220705-UNOFFICIAL-suez.zip
  1. 続けてアドオンをインストールするため、Back を選択する。

アドオンのインストール

  1. Fire HD 10 にて、Advanced > ADB Sideload を選択する。

  2. Swipe to Start Sideload をスワイプする。

  3. Ubuntu 端末にて、以下コマンドを実行し、対象デバイスに sideload と表示されていることを確認する。

    adb devices

3.Ubuntu 端末にて、以下コマンドを実行する。

cd ~/tmp/firehd10/
adb sideload 'BiTGApps-arm64-9.0.0-v3.8-ROAR[1].zip'

システムの起動

  1. Fire HD 10 にて、Reboot system now を選択する。
  2. 数分待つと、初期設定が始まるため、セットアップする。
  3. 設定が完了したら、Settings > About tablet より、Android version と LineageOS version がインストールしたバージョンになっていることを確認する。
