SoftEther VPN Server を動かしているサーバ上の Samba や SSH に VPN 経由で接続する

Published: 2024-04-14
  • #Linux
  • #Ubuntu

概要

  • Ubuntu Server に SoftEther VPN Server をインストールしている。
  • 同じサーバ上で Samba によるファイル共有サービス等を動かしている。
  • VPN 接続すると、Samba の共有ディレクトリや SSH 接続ができない。
  • Ubuntu Server にブリッジを追加し、VPN 接続したクライアントから Samba や SSH に接続できるようにする。

前提

  • Ubuntu 22.04 LTS
  • SoftEther VPN Server (Ver 4.43, Build 9799, beta)
    • ローカルブリッジ設定のブリッジ先は、物理 LAN の enp1s0 で動作している。

手順

ブリッジの追加

任意のエディタで、/etc/netplan/ 配下にある YAML ファイルを開く。

sudo vi /etc/netplan/00-installer-config.yaml

ネットワークインタフェース enp1s0 を無効化し、br0 を追加する。

今回は、192.168.11.100/24 を固定 IP として設定する。

network:
  version: 2
  ethernets:
    enp1s0:
      dhcp4: false
      dhcp6: false
  bridges:
    br0:
      interfaces:
        - enp1s0
      dhcp4: false
      dhcp6: true
      addresses:
        - 192.168.11.100/24
      routes:
        - to: default
          via: 192.168.11.1
      nameservers:
        addresses:
          - 192.168.11.1

netplan で有効化する。

$ sudo netplan apply
WARNING:root:Cannot call Open vSwitch: ovsdb-server.service is not running.

ovsdb-server.service がないため警告が出ているが、特に関係ないので無視する。

SSH 接続して作業している場合は、netplan try コマンドを使う。設定ミスで接続できなくなった場合にロールバックしてくれる。

$ sudo netplan try --timeout 30
WARNING:root:Cannot call Open vSwitch: ovsdb-server.service is not running.

念のため、サーバを再起動し、サーバと同じネットワークから以下の接続ができるか確認する。

  • SSH 接続
  • VPN 接続
  • Samba 接続

Samba に接続できなくなっていたので、次の手順で設定変更する。

Samba の設定変更

SSH 接続できるが、Samba の共有ディレクトリには接続できなくなった。

ブリッジ経由での接続が許可されていないため、設定に追加する。

任意のエディタで設定ファイルを開く。

sudo vi /etc/samba/smb.conf

interfaces に br0 を追加する。

[global]
...
# The specific set of interfaces / networks to bind to
# This can be either the interface name or an IP address/netmask;
# interface names are normally preferred
   interfaces = 127.0.0.0/8 enp1s0 br0

設定ファイルに問題がないか確認する。

$ sudo testparm
Load smb config files from /etc/samba/smb.conf
Loaded services file OK.
Weak crypto is allowed

Server role: ROLE_STANDALONE

Press enter to see a dump of your service definitions

問題なければサービスを再起動する。

sudo systemctl restart smbd && sudo systemctl restart nmbd

これで外部からブリッジ経由で Samba の共有ディレクトリに接続できるようになる。

TAP デバイスの追加

SoftEther VPN Server のローカルブリッジ設定にて、TAP デバイスを追加する。

SoftEther VPN サーバ管理マネージャを起動し、「ローカルブリッジ設定」を開く。

以下を選択し、「ローカルブリッジを追加」をクリックする。

  • 仮想 HUB: VPN (作成済みの名称)
  • 作成する種類: 新しい tap デバイスとのブリッジ接続
  • 新しい tap デバイス名: vpn (任意の名称)

TAP デバイスが追加されていることを確認する。

SoftEther VPN サーバ管理マネージャを起動し、「ローカルブリッジ設定」を開く。

ブリッジ先に enp1s0 を指定した設定を削除し、tap_vpn の設定のみが動作した状態にする。

TAP デバイスをブリッジに追加

tap_vpn をプロミスキャスモードに設定する。

sudo ip link set dev tap_vpn promisc on

tap_vpn を br0 のインタフェースに追加する。

sudo ip link set dev tap_vpn master br0

別ネットワークのクライアントから VPN 接続し、Samba の共有ディレクトリやサーバへの SSH 接続ができることを確認する。

問題なければ、VPN Server のサービス起動時に tap_vpn を br0 のインタフェースに追加する設定を入れる。

やり方は色々あると思うので、お好みで。

$ sudo cat /etc/systemd/system/vpnserver.service
[Unit]
Description = vpnserver daemon

[Service]
ExecStart = /opt/vpnserver.sh start
ExecStop = /opt/vpnserver.sh stop
ExecReload = /opt/vpnserver.sh restart
Restart = always
Type = forking

[Install]
WantedBy = multi-user.target

/opt/vpnserver.sh の中身は以下とした。

VPN Server が tap_vpn を作成したら、 br0 に追加している。

終了時は、念のため br0 から削除したうえで VPN Server を停止する。

#!/bin/bash
# chkconfig: 2345 99 01
# description: SoftEther VPN Server
DAEMON=/usr/local/vpnserver/vpnserver
LOCK=/var/lock/subsys/vpnserver

start_vpnserver() {
  $DAEMON start
  touch $LOCK
  for i in {1..10}; do
    if ip link show dev tap_vpn >/dev/null 2>&1; then
      ip link set dev tap_vpn promisc on
      ip link set dev tap_vpn master br0
      break
    else
      sleep 1
    fi
  done
}

stop_vpnserver() {
  if ip link show dev tap_vpn >/dev/null 2>&1; then
    ip link set tap_vpn promisc off
    ip link set dev tap_vpn nomaster
  fi
  $DAEMON stop
  rm $LOCK
}

test -x $DAEMON || exit 0
case "$1" in
start)
  start_vpnserver
  ;;
stop)
  stop_vpnserver
  ;;
restart)
  stop_vpnserver
  sleep 3
  start_vpnserver
  ;;
*)
  echo "Usage: $0 {start|stop|restart}"
  exit 1
  ;;
esac
exit 0

最後に、サーバを再起動し、VPN 接続経由で各種サービスが使えることを確認する。
